The **Composer** is where you create and edit posts. One draft can target multiple connected accounts, with optional per-platform overrides.

## Writing content

The main text area holds your post body. When several accounts are selected, goloom validates character limits per destination before scheduling.

### Platform-specific overrides

Use **platform tabs** to tailor content when limits or tone differ between networks (for example, a shorter Bluesky version of a Mastodon post). If you leave an override empty, the main body is used.

## Media attachments

Attach images or videos from your device or from the [Media library](/docs/guides/media-library/). For each asset you can:

- Set **alt text** for accessibility
- Reorder or remove attachments before publishing

## Live preview

The preview panel simulates how the post will look in a mobile or desktop feed on the target platform. Check previews when using media, links, or platform-specific formatting.

## Scheduling

The scheduling sidebar lets you:

- Pick a **date and time** for a one-off post
- **Add to queue** when your team uses queue slots instead of a fixed time

Validation runs before save. Fix any errors (character count, missing account, invalid media) shown inline.

## AI-generated drafts

If your workspace has AI enabled, generated content from [AI features](/docs/guides/ai-features/) can open directly in the composer. Review, edit, and schedule like any other draft.

## Permissions

- **Editors** and **owners** can create, edit, and schedule posts.
- **Viewers** can read scheduled content but cannot publish or change drafts.

See [Teams](/docs/guides/teams/) for role details.
