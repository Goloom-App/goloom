goloom ships with a locale catalog for the web UI and API error messages. The same translation files keep user-facing text consistent whether you use the browser UI or the REST API.

## Changing your language

1. Open **Settings** from the sidebar.
2. Under **Language**, pick a supported locale.
3. Most labels update immediately; reload if a cached string does not change.

Your choice is stored in browser settings and sent to the API as `Accept-Language`, so server errors match your language when available.

## Supported languages

Available locales depend on your deployment. English is always present; additional languages appear in the picker when shipped under `locales/` in the repository (for example `de.json` for German).

If a translation key is missing, the UI falls back to English.

## For contributors

Want to add or improve a language?

1. Copy `locales/en.json` to `locales/<code>.json` using a [BCP 47](https://www.rfc-editor.org/rfc/rfc5646) tag (`de`, `fr`, `pt-BR`, …).
2. Translate the **`ui`** section (interface strings) and the **`api`** section (HTTP error bodies).
3. Register the code in `frontend/src/i18n/index.ts` and, for full API coverage, in `internal/i18n/catalog.go`.
4. Run `make build` and `cd frontend && pnpm run build` to verify.

Use nested keys such as `nav.home` and `auth.signIn` rather than long flat names. Do not hardcode English in new UI code — add keys to `locales/en.json` first.

### API messages

Backend handlers use stable `snake_case` error keys (for example `invalid_json_body`) that map to the `api` section of locale files. Changing a key is a breaking change for clients that map errors by key.
