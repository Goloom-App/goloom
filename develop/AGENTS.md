# develop

## Purpose

Internal development planning documents, feature catalogs, design specs, and implementation plans. The project's planning wiki.

## Ownership

Planning documents organized by feature domain. Referenced by AGENT.md workflow during implementation.

## Local Contracts

- Plans: `plan-*.md` — implementation plans with security mandates
- Feature catalogs: `feature-catalog-*.md` — feature inventories by domain
- Security: `security-issues.md` — security tracking and mandates
- Design: `redesign.md`, `workflow-management.md` — design specs
- UI catalog: `ui-views-catalog.md` — frontend view inventory
- Parameters: `analytics-parameters.md` — analytics parameter definitions

## Work Guidance

- Plans follow structure: requirements, architecture, security, implementation steps
- Feature catalogs organized by domain: analytics, content, management, scheduling
- Security issues tracked with severity and remediation status
- Plans referenced during AGENT.md workflow Step 1 (Planning)
- Keep plans current — delete completed plans or mark as done

## Verification

No automated verification — documents reviewed manually during planning.

## Child DOX Index

None — flat directory of planning documents.
